# Privacy Policy for WebCapture Pro

**Effective Date**: January 2, 2025  
**Last Updated**: January 2, 2025

## Overview

WebCapture Pro is committed to protecting your privacy. This extension operates with a **privacy-first** approach and does not collect, store, or transmit any personal data.

## Information We DON'T Collect

WebCapture Pro does **NOT** collect:
- ❌ Personal information
- ❌ Browsing history
- ❌ Website content or screenshots
- ❌ User behavior or analytics
- ❌ Device information
- ❌ IP addresses
- ❌ Any form of tracking data

## How the Extension Works

### 100% Local Processing
- All screenshot capture happens **locally** on your device
- All PDF generation happens **offline** using client-side libraries
- **Zero** data is transmitted to any server
- **Zero** external API calls
- **Zero** cloud storage

### Permissions Explained

The extension requests the following Chrome permissions:

1. **activeTab**
   - Purpose: Capture screenshot of the currently active tab
   - Scope: Only works when you explicitly click the extension
   - Data: Screenshots are processed locally and never leave your device

2. **scripting**
   - Purpose: Inject scripts to measure page dimensions
   - Scope: Only on pages where you use the extension
   - Data: No data is stored or transmitted

3. **storage** (Optional)
   - Purpose: Save user preferences (format, quality settings)
   - Scope: All data stored locally in your browser
   - Data: Only your chosen settings, never screenshots or content

## Data Storage

### What is Stored Locally:
- User preferences (export format, quality settings)
- Extension settings

### What is NOT Stored:
- Screenshots or captured content
- Website URLs or page content
- User activity logs
- Any personal information

All stored data remains **on your device** and is **never** transmitted anywhere.

## Third-Party Services

WebCapture Pro uses **ZERO** third-party services:
- No analytics (no Google Analytics, no telemetry)
- No crash reporting
- No ad networks
- No external APIs
- No cloud services

## Offline Libraries Used

The extension uses the following **offline** JavaScript library:
- **pdf-lib**: Client-side PDF generation library
  - Purpose: Generate PDF files locally
  - Location: Bundled with extension
  - Data flow: All processing happens in your browser
